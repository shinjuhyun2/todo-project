# 1. Business purpose

## 1.1 Project background

현대인들은 학업, 과제, 개인 일정 등 다양한 일을 동시에 관리해야 하는 상황에 놓여 있다. 하지만 해야 할 일을 체계적으로 정리하지 않으면 중요한 일을 놓치거나 비효율적으로 시간을 사용하게 된다.

기존의 일정 관리 애플리케이션들은 다양한 기능을 제공하지만, 오히려 복잡한 인터페이스로 인해 간단한 작업 관리에는 불편함을 줄 수 있다. 따라서 본 프로젝트에서는 사용자가 쉽고 직관적으로 사용할 수 있는 간단한 To-Do List 프로그램을 구현하고자 한다.

특히, 기본적인 할 일 관리 기능뿐만 아니라 완료 여부 표시 및 우선순위 설정 기능을 추가하여, 사용자가 보다 효율적으로 작업을 관리할 수 있도록 하는 것을 목표로 한다. 이를 통해 프로그래밍의 기본 개념과 객체지향 설계를 적용하고, 실용적인 프로그램을 구현하고자 한다.

---

## 1.2 Goal

간단한 To-Do List 시스템을 구현하여 사용자가 할 일을 효율적으로 관리할 수 있도록 하는 것이 목표이며, 또한 객체지향 설계를 적용하여 확장 가능한 구조로 프로그램을 개발하는 것을 목표로 한다.

---

## 1.3 Target Market

기존에 있는 복잡한 어플리케이션에 불만족하거나 사용에 어려움이 있으신 분들(ex. 노인분들), 쉽게 볼 수 있으면서도 디테일한 스케줄관리를 원하시는 분들

User: 시스템을 사용하는 사용자로, 할 일을 입력하고 추가, 삭제, 완료 처리 등의 기능을 요청한다.  
System: To-Do List 프로그램 전체를 의미하며, 사용자와 상호작용하며 작업을 관리하는 역할을 한다.  
Core: 시스템의 핵심 기능을 수행하는 부분으로, 할 일의 추가, 삭제, 완료 여부 변경 등의 로직을 처리한다.  
Data: 할 일(Task) 정보를 저장하는 공간으로, 사용자로부터 입력받은 데이터와 작업 상태를 유지한다.  
Task Input / Command: 사용자가 시스템에 전달하는 요청으로, 할 일 추가, 삭제, 완료 처리 등의 명령을 포함한다.  
Task List / Status Info: 시스템이 사용자에게 제공하는 정보로, 현재 저장된 할 일 목록과 각 작업의 상태를 나타낸다.  

사용자는 시스템에 할 일과 관련된 정보를 입력하고 다양한 작업을 요청할 수 있다. 시스템은 입력된 명령을 기반으로 Core에서 로직을 처리하고, 그 결과를 Data에 저장하거나 수정한다. 또한 시스템은 Data에 저장된 할 일 정보를 조회하여 사용자에게 현재 작업 목록과 상태 정보를 제공한다.

# 4. Concept of operation

## 1) 작업 추가
Purpose  
사용자가 새로운 작업을 시스템에 추가하여 관리할 수 있도록 한다.  
Approach  
프로그램이 실행되면 사용자는 작업의 제목과 우선순위 등의 정보를 입력할 수 있다. 입력된 정보는 시스템에 전달되어 새로운 Task 객체로 생성되고 Data에 저장된다.  
Dynamics  
사용자가 작업 추가 기능을 선택하고 정보를 입력하면, 시스템은 해당 데이터를 처리하여 목록에 즉시 반영한다.  
Goals  
사용자가 새로운 작업을 쉽게 등록하고 관리할 수 있도록 한다.  

---

## 2) 작업 삭제
Purpose  
사용자가 더 이상 필요하지 않은 작업을 삭제할 수 있도록 한다.  
Approach  
사용자는 작업 목록에서 삭제할 작업을 선택하고 삭제 명령을 실행한다. 시스템은 선택된 Task를 Data에서 제거한다.  
Dynamics  
삭제 요청이 발생하면 시스템은 해당 Task를 찾아 제거하고, 변경된 목록을 사용자에게 다시 표시한다.  
Goals  
불필요한 작업을 제거하여 목록을 효율적으로 관리할 수 있도록 한다.  

---

## 3) 이미 끝난 작업, 일정 표시
Purpose  
완료된 작업과 진행 중인 작업을 구분하여 사용자에게 보여준다.  
Approach  
각 Task는 완료 여부를 나타내는 상태 값을 가지며, 시스템은 이를 기반으로 작업을 구분하여 표시한다.  
Dynamics  
사용자가 목록을 조회하면 시스템은 각 Task의 상태를 확인하여 완료 여부를 함께 출력한다.  
Goals  
사용자가 작업 진행 상황을 직관적으로 파악할 수 있도록 한다.  

---

## 4) 작업 수정
Purpose  
기존 작업의 내용을 수정할 수 있도록 한다.  
Approach  
사용자는 특정 Task를 선택한 후 제목이나 우선순위 등의 정보를 변경할 수 있다. 시스템은 변경된 내용을 Data에 반영한다.  
Dynamics  
수정 요청이 발생하면 시스템은 해당 Task를 찾아 내용을 업데이트하고 결과를 즉시 반영한다.  
Goals  
작업 정보를 유연하게 수정하여 최신 상태를 유지할 수 있도록 한다.  

---

## 5) 우선순위 결정
Purpose  
작업의 중요도를 설정하여 효율적인 작업 관리를 지원한다.  
Approach  
사용자는 Task 생성 시 또는 이후에 우선순위를 설정할 수 있으며, 시스템은 이를 저장하여 관리한다.  
Dynamics  
우선순위 변경 요청이 발생하면 시스템은 해당 Task의 priority 값을 업데이트한다.  
Goals  
작업의 중요도를 기준으로 효율적인 관리가 가능하도록 한다.  

---

## 6) 작업 검색
Purpose  
사용자가 특정 작업을 빠르게 찾을 수 있도록 한다.  
Approach  
사용자는 키워드를 입력하여 작업을 검색할 수 있으며, 시스템은 해당 키워드와 일치하는 Task를 찾아 반환한다.  
Dynamics  
검색 요청이 발생하면 시스템은 Data에 저장된 Task들을 순회하여 조건에 맞는 결과를 사용자에게 제공한다.  
Goals  
작업 탐색 시간을 줄이고 사용 편의성을 향상시킨다.  

---

## 7) 작업 정렬
Purpose  
작업을 일정 기준에 따라 정렬하여 보기 쉽게 한다.  
Approach  
사용자는 우선순위 등의 기준을 선택하여 작업을 정렬할 수 있으며, 시스템은 해당 기준에 따라 Task 목록을 재배열한다.  
Dynamics  
정렬 요청이 발생하면 시스템은 Data를 기준에 맞게 정렬하고 결과를 출력한다.  
Goals  
작업을 체계적으로 정리하여 효율적인 관리가 가능하도록 한다.

# 5. Problem statement

## Overview

My focusing time은 사용자가 자신의 할 일(Task)을 쉽고 간단하게 관리할 수 있도록 돕는 To-Do List 프로그램이다.  
사용자는 작업을 추가, 삭제, 수정하고 우선순위를 설정하여 일정을 효율적으로 관리할 수 있다.  

본 시스템은 복잡한 기능보다는 직관적이고 간단한 사용성을 목표로 하며, 기본적인 작업 관리 기능을 안정적으로 제공하는 것을 목표로 한다.  

이러한 목표를 달성하기 위해 고려해야 할 주요 문제점들은 다음과 같다.

---

## Problem #1 Task Data Management

사용자가 입력한 작업(Task) 데이터는 프로그램 내에서 저장되고 관리되어야 한다.  

데이터를 단순히 메모리에만 저장할 경우 프로그램 종료 시 모든 정보가 사라지는 문제가 발생한다.  
따라서 파일 입출력(File I/O)을 이용하여 작업 데이터를 저장하고 다시 불러올 수 있는 구조가 필요하다.  

---

## Problem #2 Input Validation

사용자가 작업을 입력할 때 잘못된 값(빈 문자열, 잘못된 우선순위 등)을 입력할 수 있다.  

이러한 잘못된 입력은 프로그램 오류를 발생시킬 수 있으므로,  
입력값에 대한 검증 로직을 구현하여 올바른 데이터만 저장되도록 해야 한다.  

---

## Problem #3 Task Search Implementation

작업의 개수가 많아질 경우 원하는 작업을 찾기 어려워질 수 있다.  

이를 해결하기 위해 간단한 문자열 기반 검색 기능을 구현할 필요가 있다.  
예를 들어, 제목에 포함된 키워드를 기준으로 작업을 필터링하는 방식이 사용될 수 있다.  

---

## Problem #4 Task Sorting Logic

사용자가 작업을 보다 효율적으로 관리하기 위해서는 정렬 기능이 필요하다.  

우선순위 또는 생성 순서를 기준으로 작업을 정렬할 수 있으며,  
이를 위해 리스트 정렬 알고리즘 또는 기본 라이브러리 정렬 기능을 활용할 수 있다.  

---

## Problem #5 Task Update Handling

작업의 내용이나 상태(완료 여부, 우선순위 등)는 언제든지 변경될 수 있다.  

따라서 특정 작업을 선택하여 해당 정보를 수정할 수 있는 기능이 필요하며,  
변경된 내용이 데이터에 정상적으로 반영되도록 구현해야 한다.
