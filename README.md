# My Focusing Time

---

## 1) 작업 추가

### Purpose
<font size="12">사용자가 새로운 작업(Task)을 시스템에 추가하여 관리할 수 있도록 한다. </font>

### Approach
프로그램이 실행되면 사용자는 작업의 제목과 우선순위 등의 정보를 입력할 수 있다. 입력된 정보는 시스템에 전달되어 새로운 Task 객체로 생성되고 Data에 저장된다.

### Dynamics
사용자가 작업 추가 기능을 선택하고 정보를 입력하면, 시스템은 해당 데이터를 처리하여 목록에 즉시 반영한다.

### Goals
사용자가 새로운 작업을 쉽게 등록하고 관리할 수 있도록 한다.

---

## 2) 작업 삭제

### Purpose
사용자가 더 이상 필요하지 않은 작업을 삭제할 수 있도록 한다.

### Approach
사용자는 작업 목록에서 삭제할 작업을 선택하고 삭제 명령을 실행한다. 시스템은 선택된 Task를 Data에서 제거한다.

### Dynamics
삭제 요청이 발생하면 시스템은 해당 Task를 찾아 제거하고, 변경된 목록을 사용자에게 다시 표시한다.

### Goals
불필요한 작업을 제거하여 목록을 효율적으로 관리할 수 있도록 한다.

---

## 3) 완료 작업 및 상태 표시

### Purpose
완료된 작업과 진행 중인 작업을 구분하여 사용자에게 보여준다.

### Approach
각 Task는 완료 여부를 나타내는 상태 값을 가지며, 시스템은 이를 기반으로 작업을 구분하여 표시한다.

### Dynamics
사용자가 목록을 조회하면 시스템은 각 Task의 상태를 확인하여 완료 여부를 함께 출력한다.

### Goals
사용자가 작업 진행 상황을 직관적으로 파악할 수 있도록 한다.

---

## 4) 작업 수정

### Purpose
기존 작업의 내용을 수정할 수 있도록 한다.

### Approach
사용자는 특정 Task를 선택한 후 제목이나 우선순위 등의 정보를 변경할 수 있다. 시스템은 변경된 내용을 Data에 반영한다.

### Dynamics
수정 요청이 발생하면 시스템은 해당 Task를 찾아 내용을 업데이트하고 결과를 즉시 반영한다.

### Goals
작업 정보를 유연하게 수정하여 최신 상태를 유지할 수 있도록 한다.

---

## 5) 우선순위 설정

### Purpose
작업의 중요도를 설정하여 효율적인 작업 관리를 지원한다.

### Approach
사용자는 Task 생성 시 또는 이후에 우선순위를 설정할 수 있으며, 시스템은 이를 저장하여 관리한다.

### Dynamics
우선순위 변경 요청이 발생하면 시스템은 해당 Task의 priority 값을 업데이트한다.

### Goals
작업의 중요도를 기준으로 효율적인 관리가 가능하도록 한다.

---

## 6) 작업 검색

### Purpose
사용자가 특정 작업을 빠르게 찾을 수 있도록 한다.

### Approach
사용자는 키워드를 입력하여 작업을 검색할 수 있으며, 시스템은 해당 키워드와 일치하는 Task를 찾아 반환한다.

### Dynamics
검색 요청이 발생하면 시스템은 Data에 저장된 Task들을 순회하여 조건에 맞는 결과를 사용자에게 제공한다.

### Goals
작업 탐색 시간을 줄이고 사용 편의성을 향상시킨다.

---

## 7) 작업 정렬

### Purpose
작업을 일정 기준에 따라 정렬하여 보기 쉽게 한다.

### Approach
사용자는 우선순위 등의 기준을 선택하여 작업을 정렬할 수 있으며, 시스템은 해당 기준에 따라 Task 목록을 재배열한다.

### Dynamics
정렬 요청이 발생하면 시스템은 Data를 기준에 맞게 정렬하고 결과를 출력한다.

### Goals
작업을 체계적으로 정리하여 효율적인 관리가 가능하도록 한다.
