# 1. Business purpose

## 1.1 Project background

현대인들은 학업, 과제, 개인 일정 등 다양한 일을 동시에 관리해야 하는 상황에 놓여 있다. 하지만 해야 할 일을 체계적으로 정리하지 않으면 중요한 일을 놓치거나 비효율적으로 시간을 사용하게 된다.  

기존의 일정 관리 애플리케이션들은 다양한 기능을 제공하지만, 오히려 복잡한 인터페이스로 인해 간단한 작업 관리에는 불편함을 줄 수 있다. 따라서 본 프로젝트에서는 사용자가 쉽고 직관적으로 사용할 수 있는 간단한 To-Do List 프로그램을 구현하고자 한다.  

특히, 기본적인 할 일 관리 기능뿐만 아니라 완료 여부 표시 및 우선순위 설정 기능을 추가하여, 사용자가 보다 효율적으로 작업을 관리할 수 있도록 하는 것을 목표로 한다. 이를 통해 프로그래밍의 기본 개념과 객체지향 설계를 적용하고, 실용적인 프로그램을 구현하고자 한다.

---

## 1.2 Goal

간단한 To-Do List 시스템을 구현하여 사용자가 할 일을 효율적으로 관리할 수 있도록 하는 것이 목표이며, 또한 객체지향 설계를 적용하여 확장 가능한 구조로 프로그램을 개발하는 것을 목표로 한다.

---

## 1.3 Target Market

기존에 있는 복잡한 어플리케이션에 불만족하거나 사용에 어려움이 있으신 분들 (ex. 노인분들), 쉽게 볼 수 있으면서도 디테일한 스케줄 관리를 원하시는 분들

**User**: 시스템을 사용하는 사용자로, 할 일을 입력하고 추가, 삭제, 완료 처리 등의 기능을 요청한다.  
**System**: To-Do List 프로그램 전체를 의미하며, 사용자와 상호작용하며 작업을 관리하는 역할을 한다.  
**Core**: 시스템의 핵심 기능을 수행하는 부분으로, 할 일의 추가, 삭제, 완료 여부 변경 등의 로직을 처리한다.  
**Data**: 할 일(Task) 정보를 저장하는 공간으로, 사용자로부터 입력받은 데이터와 작업 상태를 유지한다.  
**Task Input / Command**: 사용자가 시스템에 전달하는 요청으로, 할 일 추가, 삭제, 완료 처리 등의 명령을 포함한다.  
**Task List / Status Info**: 시스템이 사용자에게 제공하는 정보로, 현재 저장된 할 일 목록과 각 작업의 상태를 나타낸다.  

사용자는 시스템에 할 일과 관련된 정보를 입력하고 다양한 작업을 요청할 수 있다. 시스템은 입력된 명령을 기반으로 Core에서 로직을 처리하고, 그 결과를 Data에 저장하거나 수정한다. 또한 시스템은 Data에 저장된 할 일 정보를 조회하여 사용자에게 현재 작업 목록과 상태 정보를 제공한다.

# 4. Concept of operation

## 1) 작업 추가
- **Purpose**  
사용자가 새로운 작업을 시스템에 추가할 수 있도록 한다.  

- **Approach**  
사용자는 작업의 제목과 우선순위를 입력하고, 시스템은 이를 Task 객체로 생성하여 저장한다.  

- **Dynamics**  
사용자가 입력을 완료하면 시스템이 즉시 목록에 반영한다.  

- **Goals**  
작업을 쉽게 등록하고 관리할 수 있도록 한다.  

---

## 2) 작업 삭제
- **Purpose**  
불필요한 작업을 삭제할 수 있도록 한다.  

- **Approach**  
사용자가 삭제할 작업을 선택하면 시스템이 Data에서 제거한다.  

- **Dynamics**  
삭제 요청 시 즉시 목록이 갱신된다.  

- **Goals**  
작업 목록을 효율적으로 유지한다.  

---

## 3) 작업 상태 표시
- **Purpose**  
완료된 작업과 진행 중인 작업을 구분한다.  

- **Approach**  
각 Task에 상태 값을 저장하여 표시한다.  

- **Dynamics**  
조회 시 상태 정보가 함께 출력된다.  

- **Goals**  
작업 진행 상황을 쉽게 파악할 수 있도록 한다.  

---

## 4) 작업 수정
- **Purpose**  
작업 내용을 수정할 수 있도록 한다.  

- **Approach**  
사용자가 Task를 선택하여 내용을 변경하면 Data에 반영된다.  

- **Dynamics**  
수정 즉시 변경 사항이 적용된다.  

- **Goals**  
항상 최신 정보를 유지한다.  

---

## 5) 우선순위 설정
- **Purpose**  
작업의 중요도를 설정한다.  

- **Approach**  
사용자가 priority 값을 설정하고 저장한다.  

- **Dynamics**  
변경 시 즉시 반영된다.  

- **Goals**  
효율적인 작업 관리 가능  

---

## 6) 작업 검색
- **Purpose**  
작업을 빠르게 찾는다.  

- **Approach**  
키워드 기반 검색 수행  

- **Dynamics**  
조건에 맞는 Task 반환  

- **Goals**  
탐색 시간 감소  

---

## 7) 작업 정렬
- **Purpose**  
작업을 정렬하여 보기 쉽게 한다.  

- **Approach**  
기준(날짜, 우선순위 등)에 따라 정렬  

- **Dynamics**  
정렬 요청 시 재배열  

- **Goals**  
체계적인 관리

# 5. Problem statement

- 작업 관리의 비효율성 문제  
- 기존 앱의 복잡한 UI 문제  
- 데이터 저장 및 관리 구조 설계 필요  

### Non-Functional Requirements (NFRs)
- 사용성 (쉬운 UI)  
- 성능 (빠른 검색/정렬)  
- 확장성 (기능 추가 가능 구조)  
- 안정성 (데이터 유지)
